# המדריך — מה כל דבר עושה ומתי לעשות אותו

מדריך לא טכני לפריסת האתר, צעד-אחר-צעד. כל סעיף נבנה משלוש שאלות: **מה זה**, **מתי לעשות**, **איך לעשות**.

## הציור הגדול

עד עכשיו האתר חי רק על המחשב שלך — `npm run dev` מעלה אותו ב-localhost ואתה רואה אותו. אף אחד אחר לא.

"דיפלוי" אומר: לקחת את הקוד הזה, להעלות לשרת באינטרנט, ולחבר אליו את הכתובת `sharflaw.co.il` כך שכל אחד יוכל לגלוש. בנוסף — להגדיר את כל הצינורות שמסביב: מסד נתונים, שירות מייל, מעקב ביקורים, וגוגל.

הצינורות האלה הם 8 משימות נפרדות. **הסדר חשוב** — חלקן תלויות בקודמות.

```
1. Vercel deploy        ← האתר באוויר ב-URL זמני
       ↓
2. חיבור דומיין          ← sharflaw.co.il מצביע על האתר
       ↓
3. Environment Vars     ← החיבור ל-Supabase
       ↓
4. Database Migration   ← עמודה חדשה במסד הנתונים
       ↓
5. Edge Function deploy ← השרת של הטופס מתעדכן
       ↓
6. Resend domain        ← המיילים לא נופלים לספאם
       ↓
7. Google Analytics     ← לדעת מי גולש לאתר
       ↓
8. Search Console       ← לדעת איך מוצאים אותך בגוגל
```

חלקים 1–5 חיוניים לפני שהאתר באמת עובד. 6–8 אפשר אחר כך.

## 1. Vercel — לשים את האתר באוויר

### מה זה
Vercel היא חברה שמארחת אתרים. **אנלוגיה**: אם המשרד הפיזי שלך הוא הנכס, Vercel היא הקניון שמשכיר חנות בקומת קרקע. אתה דוחף את הקוד; היא נותנת URL ציבורי שכל אחד יכול לגלוש אליו.

זה הצעד הראשון כי בלעדיו אין שום מה לדבר עליו.

### מתי
- **פעם ראשונה**: בפריסה הראשונה
- **חוזר**: כל פעם שמשנים קוד ורוצים שזה יעלה

### איך
1. כניסה ל-[vercel.com/signup](https://vercel.com/signup) — אפשר עם GitHub או Google. חינם.
2. בטרמינל, מהתיקייה של הפרויקט:
   ```
   npm i -g vercel    # מתקין את הכלי (פעם אחת)
   vercel login       # נפתח דפדפן, מאשר חיבור
   vercel             # פריסה ראשונה
   ```
3. ה-CLI יחזיר URL זמני (`sharaf-law-web-abc123.vercel.app`). זה ה-**preview** — תפתח בדפדפן, וודא שהאתר נראה תקין.
4. אחרי שהפריוויו תקין:
   ```
   vercel --prod      # הופך את הגרסה לרשמית
   ```

### איך לדעת שעבד
ה-CLI מציג שני URLs (preview + prod). שניהם נטענים בדפדפן ומציגים את האתר.

### בעתיד
כל פעם שאני (או אתה) משנים קוד, להריץ `vercel --prod`. ~30 שניות. אין צורך ב-`vercel login` שוב.

## 2. חיבור דומיין

### מה זה
האתר עכשיו בכתובת ארעית. אנחנו רוצים שיהיה ב-`sharflaw.co.il`. הדומיין הזה בבעלותך אצל רשם דומיינים (Bezeqint, GoDaddy, או דומה). צריך להגיד לרשם: "כל מי שכותב sharflaw.co.il, תפנה אותו לשרת של Vercel".

**אנלוגיה**: הדומיין הוא כתובת רחוב. ה-DNS הוא דואר ישראל — מתרגם כתובות לבניינים. בלי לעדכן את הדואר, אנשים יילכו לכתובת הישנה.

### מתי
- **פעם אחת בלבד** — אחרי שהאתר חי ב-Vercel
- חזרה רק אם תחליף ספק (Vercel → Netlify) או רשם דומיינים

### איך
1. ב-[Vercel Dashboard](https://vercel.com/dashboard) → הפרויקט → Settings → **Domains**
2. **Add Domain** → להקליד `sharflaw.co.il` → Add
3. Vercel יציג רשומות DNS להוסיף, משהו כזה:
   ```
   Type: A      Name: @      Value: 76.76.21.21
   Type: CNAME  Name: www    Value: cname.vercel-dns.com
   ```
4. כניסה לפאנל הניהול אצל רשם הדומיין שלך (חברה שממנה קנית את `sharflaw.co.il`)
5. למצוא את DNS Records / DNS Management
6. להוסיף את שתי הרשומות → **שמור**
7. לחזור ל-Vercel ולחכות 5–60 דקות. Vercel ימציא תעודת SSL אוטומטית

### איך לדעת שעבד
- ב-Vercel Settings → Domains, סטטוס "Valid Configuration" עם עיגול ירוק
- בדפדפן: `https://www.sharflaw.co.il` נטען עם האתר
- מנעול ירוק (HTTPS) ליד הכתובת בדפדפן

### אם נתקעת
[whatsmydns.net](https://whatsmydns.net) → להקליד `sharflaw.co.il` → לבדוק אם הרשומה התפשטה ברחבי העולם. אם לא — לחכות עוד חצי שעה.

## 3. Environment Variables

### מה זה
האתר צריך כמה "סודות" כדי לעבוד: איפה מסד הנתונים, מה המפתחות, מה ה-GA ID. **לא** שמים אותם בקוד עצמו (כל מי שיש לו גישה לקוד יראה אותם). שמים ב-Environment Variables — מקום נסתר שרק האתר רואה.

**אנלוגיה**: כספת במשרד. הקוד הוא העובדים שלך — הם יודעים איך לפתוח את הכספת ולהוציא מסמכים, אבל המסמכים עצמם לא במגירות פתוחות.

### מתי
- **פעם אחת** עבור 2 הראשונים — אחרי הדיפלוי הראשון
- **כשתצטרף ל-GA / Search Console** עבור השאר (סעיפים 7–8)

### איך
1. Vercel Dashboard → הפרויקט → Settings → **Environment Variables**
2. **Add New**:
   - Key: `VITE_SUPABASE_URL`
   - Value: `https://dmfwoypnzerajrfowhqx.supabase.co`
   - סמן: ✓ Production, ✓ Preview, ✓ Development
3. שמור
4. חוזרים על השני: `VITE_SUPABASE_PUBLISHABLE_KEY` (הערך נמצא בקובץ `.env` המקומי)
5. **חשוב מאוד**: אחרי הוספת env var, צריך **Redeploy** מחדש. Vercel לא טוען env vars לדיפלוי קיים.
   - Dashboard → Deployments → על הפריסה האחרונה ⋯ → **Redeploy**

### איך לדעת שעבד
באתר החי, טופס יצירת קשר עובד — שולח ומקבל toast ירוק. אם לא — DevTools → Console (Cmd+Opt+I) יראה שגיאה.

### הסבר נוסף לשמות
| שם | מה זה |
|---|---|
| `VITE_SUPABASE_URL` | היכן מסד הנתונים | חובה |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | המפתח הציבורי לחיבור | חובה |
| `VITE_GA_MEASUREMENT_ID` | ID של Google Analytics | אופציונלי |
| `VITE_GOOGLE_SITE_VERIFICATION` | טוקן של Google Search Console | אופציונלי |

הקידומת `VITE_` היא דרישה של הכלי שבונה את האתר — בלעדיה הוא מתעלם מהמשתנה. תמיד להתחיל עם זה.

## 4. Database Migration

### מה זה
**אנלוגיה**: יש לך ארון תיקים במשרד. ב-Sprint 4 הוספתי הגנה מפני בוטים — חוסמים IP שמשגר 4 טפסים ביום. כדי לזכור איזה IP שלח כמה — צריכה להיות **מגירה חדשה** (עמודה בטבלה) שלא הייתה קודם.

המיגרציה היא הוראה לארון: "תוסיף מגירה חדשה בשם 'IP'". זה רץ פעם אחת.

### מתי
- **פעם אחת**, לפני שמשתמשים בקוד החדש (ה-Edge Function החדש מנסה לכתוב למגירה הזו — אם היא לא קיימת, כל הטופס שובר).
- בעתיד: כל פעם שיהיה צורך לשנות סכמה, ניצור מיגרציה חדשה.

### איך — שיטה A (הכי מהירה, דרך הדפדפן)
1. כניסה ל-[Supabase Dashboard SQL Editor](https://supabase.com/dashboard/project/dmfwoypnzerajrfowhqx/sql/new)
2. להעתיק:
   ```sql
   ALTER TABLE public.contact_submissions
     ADD COLUMN IF NOT EXISTS ip text;

   CREATE INDEX IF NOT EXISTS contact_submissions_ip_recent_idx
     ON public.contact_submissions (ip, created_at DESC);
   ```
3. **Run**
4. אם רואים `Success. No rows returned` — מעולה.

### איך — שיטה B (CLI, לטווח ארוך)
1. `brew install supabase/tap/supabase` (פעם אחת ב-Mac)
2. `supabase login` (פעם אחת — נפתח דפדפן)
3. מהשורש של הפרויקט:
   ```
   supabase link --project-ref dmfwoypnzerajrfowhqx
   supabase db push
   ```

### איך לדעת שעבד
[Dashboard → Table Editor → contact_submissions](https://supabase.com/dashboard/project/dmfwoypnzerajrfowhqx/editor) → רואים עמודה חדשה בשם `ip`.

## 5. Edge Function — לפרוס את הקוד החדש של הטופס

### מה זה
**אנלוגיה**: יש לך עוזרת משרדית שמטפלת בפניות. עד עכשיו היא הייתה הוראת עבודה ישנה: "תקבלי פנייה, תרשמי, תשלחי לי מייל". ב-Sprint 4 כתבתי לה הוראות חדשות: "**גם** תבדקי שהטלפון תקין, **גם** תזרקי בוטים, **גם** תחסמי מי שמנסה ספאם".

ההוראות החדשות אצלך בקוד אבל עדיין לא הגיעו לעוזרת. צריך לעדכן אותה — זה ה-deploy.

### מתי
- **פעם אחת** עכשיו, בגלל השינויים שלי
- **חוזר**: כל פעם שנשנה את ה-Edge Function בעתיד

### איך
דורש Supabase CLI מותקן (אם לא — שיטה B בסעיף 4):

1. אם עוד לא: `supabase login` ו-`supabase link --project-ref dmfwoypnzerajrfowhqx`
2. הפקודה היחידה: `supabase functions deploy submit-contact`

### איך לדעת שעבד
- הפלט אומר `Function deployed successfully`
- בדיקה אמיתית: שלח את הטופס באתר עם שם `<script>alert(1)</script>` — המייל שתקבל מציג את הטקסט הגולמי בלי ש-JS רץ
- שלח 4 טפסים זה אחרי זה — הרביעי מקבל הודעה "יותר מדי פניות"

## 6. Resend — שהמיילים לא יפלו לספאם

### מה זה
Resend הוא שירות ששולח את המייל אליך כשמישהו ממלא את הטופס. כרגע המייל יוצא מ-`onboarding@resend.dev` — כתובת sandbox של Resend, **לא ממשרד שרף**. Gmail/Outlook רואים מייל "מטעם משרד שרף" שיוצא מדומיין אקראי ומסווגים כספאם.

הפתרון: **לאמת** את `sharflaw.co.il` ב-Resend — להוכיח להם שהדומיין שלך — ואז לשלוח מ-`info@sharflaw.co.il`. ספקי המייל יראו ש"השולח באמת מהדומיין הזה" וישחררו.

האימות נעשה ע״י רשומות DNS אצל הרשם שלך (כמו בסעיף 2).

### מתי
- **פעם אחת**, לפני שאתה רוצה שהמיילים יגיעו לתיבה ולא לספאם
- אופציונלי לתחילה — אבל ברגע שמתחילים לקבל פניות אמיתיות, חיוני

### איך
1. כניסה ל-[Resend.com](https://resend.com) → להירשם (חינם עד 3,000 מיילים בחודש)
2. Resend Dashboard → **Domains** → **Add Domain** → `sharflaw.co.il`
3. Resend יציג 3 רשומות DNS להוסיף — TXT עבור SPF, DKIM, DMARC. נראה כך:
   ```
   Type: TXT   Name: send                  Value: "v=spf1 include:..."
   Type: TXT   Name: resend._domainkey     Value: "p=MII..."
   Type: TXT   Name: _dmarc                Value: "v=DMARC1; p=none;"
   ```
4. **כמו בסעיף 2** — להוסיף את הרשומות אצל רשם הדומיין שלך
5. לחזור ל-Resend ולחכות 5–60 דקות. אחרי הזמן, כל הרשומות יראו ✓ ירוק
6. ב-Resend → **API Keys** → **Create API Key** → להעתיק את המפתח (מתחיל ב-`re_...`)
7. ב-[Supabase → Edge Functions → Settings](https://supabase.com/dashboard/project/dmfwoypnzerajrfowhqx/settings/functions):
   - `RESEND_API_KEY` = המפתח שהעתקת
   - `RESEND_FROM` = `info@sharflaw.co.il` (או `noreply@sharflaw.co.il`)
8. אחרי הוספת ה-env vars, לפרוס מחדש את ה-Edge Function (סעיף 5).

### איך לדעת שעבד
שלח טופס לבדיקה דרך האתר. המייל מגיע ל-`sharflaw@bezeqint.net` — **בתיבה הראשית, לא בספאם**. השולח מופיע כ-`info@sharflaw.co.il`.

## 7. Google Analytics

### מה זה
GA הוא כלי מעקב — אומר לך כמה אנשים גלשו לאתר היום, מאיפה הגיעו (Google? פייסבוק?), כמה זמן נשארו, באיזה עמודים ביקרו. **לא** רואים פרטים אישיים (שמות, טלפונים), רק סטטיסטיקות מצטברות.

חשוב לעו״ד: לדעת מאיפה לקוחות מגיעים, איזה עמודים מעניינים, אם יש ביקושים מ-Google ל"עורך דין מקרקעין בת ים".

### מתי
- **פעם אחת**, רצוי אחרי שהאתר חי בדומיין הסופי

### איך
1. כניסה ל-[analytics.google.com](https://analytics.google.com) עם חשבון Google שלך
2. **Admin** (גלגל שיניים בפינה למטה) → **Create** → **Account**:
   - Account name: "Sharflaw"
   - מסמנים את כל הצ'קבוקסים → **Next**
3. **Create Property**:
   - Property name: "משרד עו״ד שרף"
   - Time zone: Israel
   - Currency: ILS
4. **Next** → Business details → "Other" / "Legal services" → **Create**
5. בוחרים **Web** → URL: `https://www.sharflaw.co.il` → Stream name: "Web" → **Create stream**
6. בעמוד שנפתח: **Measurement ID** — מספר בפורמט `G-XXXXXXXXXX`. **להעתיק.**
7. ב-Vercel → Settings → Environment Variables:
   - `VITE_GA_MEASUREMENT_ID` = `G-XXXXXXXXXX`
8. Vercel → Deployments → ⋯ → **Redeploy**
9. אחרי 2 דקות, פותחים את האתר בדפדפן, מאשרים את באנר העוגיות
10. במקביל [GA → Realtime](https://analytics.google.com) צריך להציג נקודה אחת — אתה כרגע גולש

### איך לדעת שעבד
GA → Realtime מציג ביקורים. אחרי יומיים, GA → Reports → Acquisition יציג איפה אנשים מצאו את האתר.

### הערה חשובה
GA נטען רק אחרי שהמשתמש מאשר את באנר העוגיות. אנשים שדוחים — לא נספרים. זה דרישה של מדיניות הפרטיות שכתבתי, וזה גם הרוח של החוק הישראלי.

## 8. Google Search Console

### מה זה
Search Console הוא לוח בקרה שגוגל נותנים לבעלי אתרים — מראה איזה חיפושים הוליכו אליך, באיזה תוצאה אתה מופיע, אילו מילות מפתח מביאות תנועה.

ההבדל מ-GA: **GA מראה מה קורה באתר**, **Search Console מראה מה קורה ב-Google לפני שמגיעים אליך**.

חשוב לעו״ד מקרקעין: אם אתה מופיע במקום 12 ב-"עורך דין מקרקעין בת ים" — Search Console יגיד לך, ותחליט אם להפיק תוכן שיקפיץ אותך.

לפני שגוגל מציג נתונים, הוא דורש הוכחה שאתה הבעלים של הדומיין. שתי שיטות אימות; DNS עדיפה.

### מתי
- **פעם אחת**, ככל שמוקדם יותר אחרי שהאתר חי

### איך — שיטת DNS (מומלצת)
1. כניסה ל-[search.google.com/search-console](https://search.google.com/search-console)
2. **Add property** → **Domain** (העמודה השמאלית) → `sharflaw.co.il` → **Continue**
3. גוגל מציג רשומת TXT לאימות:
   ```
   google-site-verification=Ab1Cd2...
   ```
4. אצל רשם הדומיין → DNS → להוסיף רשומה:
   ```
   Type: TXT   Name: @ (או ריק)   Value: google-site-verification=Ab1Cd2...
   ```
5. לחכות 5–60 דקות → לחזור ל-Search Console → **Verify**
6. אחרי האימות → **Sitemaps** בתפריט הצד → **Add a new sitemap** → `https://www.sharflaw.co.il/sitemap.xml`

### איך — שיטת HTML (אם DNS לא נוח)
1. במקום **Domain** בוחרים **URL prefix** → `https://www.sharflaw.co.il/`
2. בוחרים **HTML tag** → גוגל נותן ערך כמו `Ab1Cd2Ef3...`
3. ב-Vercel → Settings → Environment Variables:
   - `VITE_GOOGLE_SITE_VERIFICATION` = `Ab1Cd2Ef3...` (רק הערך)
4. Vercel → Deployments → Redeploy
5. חזרה ל-Search Console → **Verify**

### איך לדעת שעבד
- כפתור Verify הופך לירוק
- אחרי 1–7 ימים, Search Console → Performance מתחיל להציג נתונים
- Search Console → Pages רואים את `/`, `/privacy`, `/accessibility` כעמודים שגוגל סרק

## אם נתקעת — שלוש בעיות נפוצות

### 1. "DNS לא מתעדכן"
DNS לוקח עד שעה. תבדוק ב-[whatsmydns.net](https://whatsmydns.net) — מקליד את הדומיין, בוחר את סוג הרשומה (TXT/A/CNAME), ורואה אם הרשומה התפשטה ברחבי העולם. אם יש הרבה ✗ אדומים — לחכות עוד חצי שעה.

### 2. "שיניתי env var ב-Vercel ולא רואים שינוי"
Vercel לא טוען מחדש env vars אוטומטית. **חייבים Redeploy**: Dashboard → Deployments → על הפריסה האחרונה ⋯ → **Redeploy**. ~30 שניות.

### 3. "הערך לא נכון" / "השוואה נכשלה"
קופי-פייסט הוא חבר אבל גם אויב. בדוק:
- יש רווחים מיותרים בתחילת/סוף הערך?
- העתקת גרשיים `"..."` במקום הערך עצמו?
- שמת את הטוקן הנכון? (Search Console נותן שני טוקנים — אחד ל-DNS ואחד ל-HTML; לא להחליף).

# דו"ח חילוץ — תשתית לעיצוב מחדש

מסמך זה מחלץ את מצב הקוד הנוכחי של אתר משרד עו״ד שרף. אין כאן המלצות, שיפוטים או הצעות שיפור — רק עובדות מהקוד.

---

## 1. מחסנית טכנולוגית

- **Framework**: Vite 5.4 + React 18.3 (SPA, ללא SSR/SSG)
- **שפה**: TypeScript 5.5 עם `strict: true`
- **Styling**: Tailwind CSS 3.4 (utility-first), עם `tailwindcss-animate` ו-`@tailwindcss/typography`. אין CSS Modules, אין styled-components, אין SCSS. סטיילים inline דרך `className` בלבד.
- **ספריית קומפוננטות**: shadcn/ui — אך נוקה. נשארו רק 4 רכיבי UI מתוך 47 שהיו במקור: `Toaster` (`@radix-ui/react-toast`), `Sonner` (`sonner`), `TooltipProvider` (`@radix-ui/react-tooltip`), ו-`use-toast` hook.
- **Routing**: `react-router-dom` 6.26 (BrowserRouter, SPA routing)
- **State/Data**: `@tanstack/react-query` 5.56 — מותקן ומעוטף ב-`QueryClientProvider` ב-App, אך **לא בשימוש בפועל** באף קומפוננטה.
- **Backend**: Supabase — `@supabase/supabase-js` 2.105. כולל Edge Function אחת (Deno) ב-`supabase/functions/submit-contact/`.
- **Icons**: `lucide-react` בלבד. אין SVG משלנו, אין set אחר.
- **Package manager**: npm (`package-lock.json`). אין `bun.lock` או `yarn.lock`.
- **Testing**: Vitest 4.1 + `@testing-library/react` + jsdom. 39 בדיקות עוברות.
- **Linting/Build**: ESLint 9.9 + typescript-eslint 8.0; build = `tsc --noEmit && vite build`.
- **יעד פריסה**: Vercel — `vercel.json` בשורש מגדיר rewrites ל-SPA fallback ו-security headers (CSP, HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy). לא דופלאי בפועל עדיין.
- **קבצי קונפיגורציה**:
  - `vite.config.ts` — alias `@` → `./src`, port 8080, plugin משלנו ל-Search Console meta injection
  - `tailwind.config.ts` — extends colors/borderRadius/keyframes/animations, container max 1400px עם padding 2rem, dark mode `["class"]`
  - `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json` — strict mode, paths alias
  - `eslint.config.js` — flat config
  - `vitest.config.ts` — environment jsdom, setup ב-`src/test/setup.ts`
  - `.env.example` עם 4 משתנים: `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `VITE_GA_MEASUREMENT_ID`, `VITE_GOOGLE_SITE_VERIFICATION`
  - `vercel.json` עם CSP מהודק (כולל allowlist של Supabase host ספציפי, Google Fonts, Google Analytics, Google Maps frames)
  - `postcss.config.js`, `components.json` (shadcn config)

## 2. ארכיטקטורת דפים

האתר הוא SPA עם 4 routes:

### `/` (Index)
- **שם בקוד**: `src/pages/Index.tsx`
- **מטרה עסקית**: עמוד נחיתה יחיד. כל התוכן השיווקי, השירותים, ההמלצות, המידע על המשרד והטופס — באותו URL.
- **סדר הסקציות**:
  1. **Header** (sticky top, מחוץ ל-`<main>`)
  2. **Hero** — `id="home"`
  3. **WhyChooseUs** — אין id (אין anchor אליה)
  4. **Services** — `id="services"`
  5. **Reviews** — `id="reviews"`
  6. **About** — `id="about"`
  7. **BuyingGuide** — `id="articles"` (האנקור בנייוויגציה הוא "מאמרים" — שם מטעה, יש רק מדריך אחד)
  8. **ContactForm** — `id="contact"`
  9. **Footer** (מחוץ ל-`<main>`)
  10. **WhatsAppFloat** (fixed bottom-left, מחוץ ל-`<main>`)
  11. **CookieBanner** (fixed bottom, רק כאשר `VITE_GA_MEASUREMENT_ID` מוגדר)

### `/privacy`
- **שם בקוד**: `src/pages/Privacy.tsx`
- **מטרה**: מדיניות פרטיות לפי חוק הגנת הפרטיות תשמ״א-1981
- **מבנה** (כולו בתוך `<LegalLayout>`):
  1. Header פשוט (לוגו + שם המשרד, קישור הביתה)
  2. כפתור "חזרה לדף הבית"
  3. כותרת "מדיניות פרטיות" + תאריך עדכון
  4. 8 סעיפים ממוספרים (סוגי מידע, שימוש, צדדים שלישיים, אבטחה, שמירה, זכויות, שינויים, יצירת קשר) + תת-סעיף עוגיות
  5. Footer רגיל

### `/accessibility`
- **שם בקוד**: `src/pages/Accessibility.tsx`
- **מטרה**: הצהרת נגישות לפי תקנות שוויון זכויות לאנשים עם מוגבלות תשע״ג-2013
- **מבנה**:
  1. אותו Header פשוט של LegalLayout
  2. כותרת + תאריך
  3. 7 סעיפים: מחויבות, רמת תאימות, אמצעים קיימים, מגבלות ידועות, נגישות פיזית, פנייה, רכז נגישות

### `*` (NotFound)
- **שם בקוד**: `src/pages/NotFound.tsx`
- **מטרה**: 404 catch-all
- **מבנה**: עמוד מינימלי במרכז המסך — "404", "העמוד לא נמצא", הודעת הסבר, כפתור "חזרה לעמוד הבית"

- **Tailwind utility בשימוש**: `space-x-reverse` במספר מקומות (Header nav, Footer social).

### שפות נתמכות בפועל

- **עברית בלבד**. כל הטקסטים, התוויות, הודעות השגיאה, ה-aria-labels — בעברית.
- ב-Schema.org JSON-LD יש הצהרה `knowsLanguage: ["he", "en"]` ו-`availableLanguage: ["Hebrew", "English"]` — אך אין שום תוכן באנגלית באתר.
- `<title>` באנגלית-במחצית: שם בעברית.

### Workarounds / hacks ב-RTL

1. **`.container { direction: rtl }`** ב-`src/index.css` — דריסה גלובלית של utility container של Tailwind. כל container באתר מקבל direction מפורש, גם אם נשתמש בו עתידית עבור תוכן LTR (למשל סקציה באנגלית) — תהיה התנגשות.

2. **שימוש מעורב ב-spacing primitives**:
   - חלק מהקומפוננטות משתמשות ב-`gap-X` (modern, RTL-aware אוטומטית): Header logo flex, Hero buttons, ContactForm fields
   - חלק משתמשות ב-`space-x-X space-x-reverse` (older approach): Header nav, Footer social
   - חלק משתמשות ב-`ml-3` או `mr-3` ידני: BuyingGuide icons, ContactForm icons
   - אין כלל אחיד.

3. **`text-right` במובייל nav** ב-Header — מיותר כי `dir="rtl"` כבר מגדיר זאת. רומז שמישהו לא סמך על ירושת ה-direction.

4. **`flex-row` במקום `flex-row-reverse` בכל מקום** — Tailwind חכמה מספיק עם `gap-` ו-`flex` ב-RTL, אז זה תקין. אבל בכמה מקומות יש `space-x-reverse` שמתבסס על `flex-row` קלאסי — אינקונסיסטנטי.

5. **כפתור WhatsAppFloat ממוקם `left-6`** — כלומר בצד שמאל פיזית. במצב RTL, "התחלה" היא ימין, "סוף" שמאל. הצבת הכפתור ב-`left-6` משאירה אותו צד שמאל מבלי להתחשב בכיוון ה-document. אם תוכן עברי, הכפתור יוצא ב"צד ההפוך" (פינה שמאלית-תחתונה) — בכוונה או לא.

6. **og-image עם RTL** — נוצר עם `python-bidi` כדי לטפל בעברית. מעורב עם ספרות לטיניות (`1964`). הקובץ עצמו אינו text-html, אז אין לזה השלכה דינמית, אבל ההכנה דרשה bidi processing.

7. **ה-iframe של Google Maps** — מקבל `referrerPolicy="no-referrer-when-downgrade"` ולא מכוון מפורשות ל-RTL. גוגל פייס מטפל בפנים.
